## 💬 Commit Message Guidelines

Use **clear and meaningful commit messages** following this format:

### Format
```
[Name] <Action>: <commit message>
```

### Actions
- **Add** - Adding new features or files
- **Refactor** - Restructuring existing code
- **Fix** - Fixing bugs or issues
- **Update** - Updating or modifying existing code

### Examples
```
[Ayush Sharma] Add: Programming elements assignments
[Ayush Sharma] Refactor: Optimized control flow loops
[Ayush Sharma] Fix: Bug in PalindromeChecker logic
[Ayush Sharma] Update: Inheritance hierarchy implementation
```

### ❌ Avoid
```
update
done
changes
final
fixed stuff
```

## 🔄 Git Workflow

Follow these steps for every commit:

### Step 1: Pull Latest Changes
```bash
git pull origin <branch-name>
```

### Step 2: Add Files
```bash
git add .
```

### Step 3: Commit Changes
```bash
git commit -m "[Your Name] Add: Your meaningful commit message"
```

### Step 4: Push Changes
```bash
git push origin <branch-name>
```

### Complete Example
```bash
# Switch to core-java-practice branch
git checkout core-java-practice

# Pull latest changes
git pull origin core-java-practice

# Make your changes...

# Add files
git add .

# Commit with meaningful message
git commit -m "[Ayush Sharma] Add: Programming elements and control flow assignments"

# Push to repository
git push origin core-java-practice
```

## ⚡ Quick Tips

1. **Always pull before pushing** to avoid merge conflicts
2. **Use meaningful commit messages** - they help in code reviews
3. **Follow the folder structure strictly** - makes code organization easier
4. **Keep your branch updated** with the latest changes
5. **One feature per commit** - makes reverting easier if needed
